# Прогнозирование покупок в интернет-магазине

 📋 Описание проекта

Разработка модели машинного обучения для предсказания вероятности совершения покупки клиентом в течение следующих 90 дней на основе истории покупок и взаимодействия с рекламными рассылками.

🎯 Задачи

- Исследование и предобработка данных о покупках и рассылках
- Разработка полезных признаков (feature engineering)
- Создание модели бинарной классификации
- Оптимизация модели и максимизация метрики ROC-AUC
- Интерпретация результатов и анализ важности признаков

📊 Данные


Используются следующие источники данных:

| Файл | Описание | Размерность |
|------|----------|-------------|
| `apparel-purchases` | История покупок клиентов (товары, цены, категории) | 202 208 строк × 6 колонок |
| `apparel-messages` | История рекламных рассылок (события, каналы) | 12 739 798 строк × 7 колонок |
| `apparel-target_binary` | Целевая переменная (покупка в течение 90 дней) | 49 849 клиентов × 2 колонки |
| `full_campaign_daily_event` | Агрегированная статистика по рассылкам | 131 072 строки × 24 колонки |
| `full_campaign_daily_event_channel` | Агрегированная статистика по каналам | 131 072 строки × 36 колонок |

🔍 Основные этапы работы

1. Предобработка данных
- Преобразование типов данных (даты → datetime)
- Удаление дубликатов:
  - `purchases`: удалено 73 020 строк (36%)
  - `messages`: удалено 48 610 строк (0.4%)
- Обработка категориальных признаков:
  - Создание бинарных признаков для 8 категорий первого уровня
  - Удаление 2.6% строк с некорректными категориями
- Агрегация данных по клиентам:
  - 49 849 клиентов × 34 признака
- Обработка пропусков:
  - Заполнение нулями (отсутствие событий = 0)
  - Создание индикатора `has_bulk_campaign` для клиентов без рассылок (15.8%)

2. Исследовательский анализ данных (EDA)
- Дисбаланс классов: только 1.93% клиентов совершили покупку
- Клиенты без рассылок: 15.8% не имеют истории взаимодействия
- Ключевые различия:  клиенты с target=1 тратят больше, покупают чаще, активнее взаимодействуют с рассылками
- Мультиколлинеарность: выявлены пары признаков с корреляцией >0.95, удалено 4 признака

3. Моделирование
- Рассмотрены алгоритмы: `RandomForest`, `XGBoost`, `CatBoost`
- Поиск гиперпараметров: `RandomizedSearchCV` (5-fold CV, ROC-AUC, 100 итераций)
- Различные варианты масштабирования: `StandardScaler`, `MinMaxScaler`, `RobustScaler`, `passthrough`

4. Результаты

Лучшая модель: `XGBClassifier` с `RobustScaler`

Оптимальные гиперпараметры:
- `subsample`: 0.8
- `scale_pos_weight`: 2
- `n_estimators`: 50
- `max_depth`: 3
- `learning_rate`: 0.1
- `colsample_bytree`: 0.6

Метрики:
- ROC-AUC (кросс-валидация) - 0.7217
- ROC-AUC (тестовая выборка) - 0.7405

5. Анализ важности признаков

Топ-5 признаков по Permutation Importance:

- `unique_purchase_days` - Количество дней с покупками 
- `event_click` - Количество кликов по рассылкам 
- `conv_send_to_open` - Конверсия из отправки в открытие 
- `price` - Средняя цена товара (отрицательное влияние) 
- `channel_mobile_push` - Активность в мобильном канале 

## 📈 Ключевые выводы

1. Лучший алгоритм: XGBoost показал наилучший результат (ROC-AUC 0.7405)
2. Проблема дисбаланса: модель отлично определяет некупивших (99.98%), но не обнаруживает покупателей (0% recall) — типичная ситуация для сильно несбалансированных данных
3. Ключевые факторы покупки:
   - Покупательская активность (дни с покупками, клики)
   - Вовлеченность в рассылки (конверсии, мобильный канал)
   - Ценовые характеристики (обратная связь)

💡 Рекомендации для бизнеса

- Фокус на вовлеченных клиентах — высокая историческая активность = высокий потенциал
- Развитие мобильного канала — push-уведомления эффективны для стимулирования покупок
- Ценовые акции — скидки могут повысить конверсию (обратная связь с ценой)
- Вовлечение "холодной" аудитории — 15.8% клиентов без рассылок — потенциал для роста

 🔧 Рекомендации для улучшения модели

- Применить методы борьбы с дисбалансом (SMOTE, undersampling)
- Настроить порог классификации для оптимизации recall
- Рассмотреть ансамблирование моделей
